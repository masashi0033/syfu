# MANEKINEKO BREED SIM 仕様書
**SyFu Season 1 ブリード収支シミュレータ（breed.html）**
初心者向け解説版｜2026年7月版

---

## 1. このツールは何をするもの？

SyFuの「ブリード（繁殖）」機能で、**招き猫NFTを掛け合わせて新しいNFTを作るときのコストと確率**を計算するツールです。

ブリードは「EVT・Econ・鯛焼き（TAIYAKI）」という3種類の通貨・アイテムを消費し、確率でレアな子どもが産まれます。「レアが出るまで平均いくらかかるのか」「運が悪いと最悪いくらかかるのか」を、ゲームを始める前に机上で確認できます。

ブラウザで開くだけで動きます（インストール不要・通信不要・スマホ対応）。

### 最低限おぼえる用語

| 用語 | 意味 |
|---|---|
| MANEKINEKO | SyFuの招き猫NFT。ブリードの「親」になる |
| Uncommon / Rare | NFTのレア度。Rareのほうが希少で高価 |
| EVT | 換金できるトークン。ブリードの主な支払い手段 |
| Econ | ゲーム内通貨。ブリードのたびに消費 |
| TAIYAKI（鯛焼き） | ブリードに必要なアイテム。Uncommon用・Rare用・White（Elite抽選用）がある |
| Breed Count（BC） | その親が何回ブリードしたか。**1体最大5回**で打ち止め |
| P-rank | 個体の性能ランク（P0〜P4）。高いほど良い |
| Prime | 成長の最終段階。**親はPrimeでないとブリード不可** |
| 血統（Z/L/V/N） | 4つの系統グループ。相性（Nicks）の判定に使う |
| Nicks | 血統の相性ボーナス。子のP-rankが上振れしやすくなる |
| Inbreeding | 同じ個体が家系図の両側に重複すること。位置によって良い効果にも悪い効果にもなる |
| Origin / Elite | 特別な個体。家系図でうまく重複させるとRare抽選が発生する |

## 2. 画面構成（5つのタブ）

### タブ01：コスト計算
**「この2体を掛け合わせたら、1回いくら？」を出す電卓です。**

- 入力：組み合わせ（U×U／左U×右R／左R×右U／R×R）、左右の親のBreed Count、White鯛焼きの使用有無
- 出力：EVT／Econ／鯛焼きの消費量、Rareが出る確率、Elite抽選率

ポイント：**Season 1ではブリード成功ごとにRight Scroll（採掘権NFT）が1個付与**されます（店舗プール解放に使用）。また**Econは親のBCが増えるほど高くなります**（1につき+20）。また左右は対等ではなく、Rareを左に置くと右に置くより高くつきます（左Rare+40／右Rare+20）。

### タブ02：Rare獲得試算
**「Rareが出るまで合計いくら？」を運の良し悪し3パターンで出します。**

| シナリオ | 意味 |
|---|---|
| 上振れ | 1回目で当たった場合 |
| 期待値 | 平均的な回数（確率50%なら2回、20%なら5回、10%なら10回） |
| 下振れ（90%tile） | 「90%の人はここまでに当たる」回数。運が悪い側の目安 |

グラフは「n回目までに当たっている確率」の曲線です。赤い点線（90%ライン）に達する回数が下振れの目安。

ポイント：親は5回で打ち止めなので、**確率10%・20%の組み合わせは当たる前に親が枯れやすい**ことが表から読み取れます。Rareを増やす目的ならR×R（50%）一択です。

### タブ03：実践ガチャ
**実際にボタンを押して1回ずつ引く体験モードです。**

まり玉が白（外れ＝Uncommon）か金（当たり＝Rare）に変わり、消費リソース・親の交代・累計が自動記帳されます。確率の「下振れ」がどれくらい起こるかを体感するためのタブです。※乱数はツール内のもので、実際のゲーム結果とは無関係です。

### タブ04：1体目Golden
**「Rareを1体も持っていない人」がRareを狙う唯一の計算可能ルートの設計図です。**

Rareの固定抽選（50/20/10%）は「Rareの親」が必要なので、最初の1体には使えません。代わりに、Origin（Genesis血統の特別個体）を家系図の**3世代目×4世代目（Golden）**に重ねると、Rareの上振れ抽選が発生します（当選率は公式非公開）。

- 入力：引きたい抽選回数、White鯛焼き併用の有無
- 出力：必要な総ブリード数／EVT／Econ／鯛焼き／**Prime育成が必要な体数**／必要Origin数、および家系図の図解

重要ルール（ツール内にも警告表示）：
1. **2本の家系図チェーンはOrigin直下から完全に別系統にする**こと。途中の個体を共有すると浅い世代で重複が起きて「Forbidden」判定になり、抽選ごと消えます。
2. 右親に使うUncommonもチェーン間で使い回し不可。
3. 完成した「深いペア」は双方BC0なので**同じペアで最大5回**抽選を引き直せます。
4. Origin1体（5枠）で作れるのは2ペア＝抽選10回まで。
5. 本当のボトルネックはお金ではなく**各世代をPrime（Lv10）まで育てる時間**です。

### タブ05：手持ち最適化
**自分のNFTを登録すると「目的別にどのペアで交配すべきか」をランキング表示します。**

NFT1体ごとに登録する項目：

| 項目 | 説明 |
|---|---|
| 個体名 | 自由入力。**同じ名前＝同一個体**としてInbreeding照合に使われる |
| レア度／特殊／P-rank | Uncommon・Rare／なし・Origin・Elite／P0〜P4 |
| Breed Count | 0〜5（5は交配不可） |
| ブリード可 | Prime・寿命40%以上・HP100%・出品/クールダウン外か |
| 血統系統（自身＋1G〜3G） | 家系図の系統グループ（Z/L/V/N）を4世代分 |
| Genesis血統の位置 | Genesis血統を持つ場合、その位置 |
| 注目個体 | 家系図に入っているOrigin/Elite/重複しうる個体（名前・世代・種別） |

目的は6種類から選択：

| 目的 | 何を優先する？ |
|---|---|
| Rare獲得 | 固定抽選率（R×R 50% ＞ 左R 20% ＞ 右R 10%） |
| 高P-rank | Nicks数（最大3）＋Genesis premium＋左親P2以上 |
| Good Inbreeding | Origin/Eliteが3G×4G（Golden）か4G×4G（Stable）で重複するペアのみ |
| Special Risk | 特殊個体が浅い世代で重複する低確率の博打（確率非公開） |
| Elite生成 | U×U＋White鯛焼き。Eliteは左親のP-rankを継承するので左親のP-rank重視 |
| コスト最小 | EVT＋Econが最小のペア |

結果カードには順位・コスト・判定チップ（Rare率/Nicks/GOLDEN/FORBIDDENなど）・推薦理由が表示されます。**同じ2体でも左右を入れ替えて両方評価**します（コスト・確率・血統継承すべて左右で変わるため）。

## 3. 計算式（かみ砕き版）

すべてSyFu公式GitBook第9章（Season 1）の数値です。

**1回のコスト**
- EVT：U×U=100／左U右R=120／左R右U=140／R×R=160（固定）
- Econ：左親分＋右親分。各親＝基本値（Uncommon 50、左Rare 90、右Rare 70）＋BC×20
- 鯛焼き：U×U=Unc2枚／片Rare=Unc1+Rare1／R×R=Rare2枚

**Rare固定抽選率**：R×R=50%／左R×右U=20%／左U×右R=10%（外れはUncommon P2以上）

**試行回数**
- 期待値＝1÷確率
- 下振れ＝「90%の人が当たるまでの回数」＝log(0.1)÷log(1−確率) を切り上げ

**Inbreeding判定（子から見た世代で判定）**
親そのものは子の1世代目、親の1G祖先は子の2世代目…と数えます。

| 重複位置 | 判定 | 効果 |
|---|---|---|
| 4G×4G | Stable | 特殊個体なら上振れ抽選 |
| 3G×4G | Golden | 特殊個体ならより強い上振れ抽選 |
| 3G×3G、2G×4G | Danger | Nicks・Genesis premium無効。特殊個体ならSpecial Risk |
| 2G×2G、2G×3G、1G×どこでも | Forbidden | 同上（最も危険） |

**Nicks**：左親の系統→右親の系統が「Z→L／L→V／V→N／N→Z」なら成立。左右4世代×4世代＝最大16候補から**有効3つまで**採用。同系統はNeutral（Season 1では効果なし）、それ以外はNone。

**P-rank下限ルール**（公式明記）
| 条件 | 子のP-rank下限 |
|---|---|
| 有効Nicks3つ ＋ 左親P2以上 | **P1** |
| Rare入りブリードでRare抽選に外れた場合 | Uncommon **P2以上** |
| White鯛焼きでElite当選 | 左親のP-rank以上 |
| Elite個体を左親に使用 | 最大3体の子がそのEliteのP-rankを継承 |

**Season 1の全32血統**
| 群 | 血統名 |
|---|---|
| [Z] Zenith | Genesis / Chronos / Nexus / Logos / Vector / Aura / Apex / Equinox |
| [L] Luna | Cobalt / Titanium / Nickel / Palladium / Rhodium / Iridium / Osmium / Chrome |
| [V] Vega | Jasper / Onyx / Zircon / Spinel / Garnet / Topaz / Beryl / Citrine |
| [N] Nebula | Basalt / Granite / Slate / Shale / Gneiss / Marble / Obsidian / Andesite |

Pedigree表記は `[L] Cobalt-8e2A`（群記号＋血統名＋個体識別コード）。**Inbreeding判定は個体識別コード（tokenId）で行う**ため、同じ血統名でもコードが違えば別個体です。OGは標準4群に属さず `[OG] Primal-02A7` と表示され、OG個体自体はNicks判定に使われません（左に置くと31血統から新血統が発現）。

## 4. 注意事項

- Golden/Special Riskの当選率、P-rank上振れ幅は**公式非公開**。ツールが保証できるのは「抽選を引く権利のコスト」までです。
- 仕様はSeason 1時点のもの（2026年7月のGitBook再編に照合済み。ブリードのコア数値＝EVT 100/120/140/160・Rare率50/20/10%・Econ表・Inbreeding表に変更なし）。公式のアップデートで数値が変わったら結果は当てになりません。
- タブ05の家系図は「全祖先」ではなく「注目個体だけ登録」する簡易方式です。登録漏れがあるとInbreeding判定も漏れます。
- 登録データは保存されません（画面を閉じると消えます）。
- 本ツールは収益・当選を保証しません。投資は自己責任で。
